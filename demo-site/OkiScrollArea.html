OkiScrollArea is solution for scrolling any kind of content without using browser's scroll bars. This OkiWidget can be also controlled by touch so you can use it on mobile devices.

Basic usage

Important - OkiScrollArea ommits css width and height. Widget works on dimension passed during initialization.
If you really need to set dimension using only css you should wrap OkiScrollArea div by another div and set dimension to wrapper (by default viewport is set to 100% so scroll area gets wraper size)
NOTE: Causion with images without fixed dimensions inside scroll area. If they loads after widget initialization you will get wrong dimension. You can fix it by move initialization code to $(window).load(), add fixed dimensions or call resize method from OkiScrollArea API after $(window).load().

Viewport and content settings

You can controll initial dimension of scroll area. Use viewportWidth/viewportHeight and contentWidth/contentHeight variables.

Note - if you set 'contentWidth' above 'viewportWidth' OkiScrollArea automatically turn on horizontal scrollBar.

After parent resize you should call resize() method for API
This works only if viewportWidth/viewportHeight is set to 100%.
 
api.softDetachOn

This method activates events prevented by OkiScrollArea. It can be helpfull when you need to disable OkiScrollArea for some time.
 
api.softDetachOff

This method turns on all OkiScrollArea's functionality. It can be helpfull when you need to disable OkiScrollArea for some time and reenable it.
 

Note that after all api call you should always call api.resize() at the end. This comes from performance reasons (you can set many dimension settings and after all call api.resize() to see changes).
